# Architecture Decision Records (ADR)

## ADR-001: Docker Compose вместо Terraform

**Статус:** Принято  
**Контекст:** Проект разворачивается локально на Ubuntu VM.  
**Решение:** Использовать Docker Compose для оркестрации всех сервисов.  
**Обоснование:** Упрощает локальную разработку, не требует облачного аккаунта, воспроизводимо.

---

## ADR-002: MinIO как S3-совместимое хранилище

**Статус:** Принято  
**Решение:** MinIO вместо AWS S3.  
**Обоснование:** Полностью совместим с S3 API (boto3), работает локально, поддерживает multi-bucket.

---

## ADR-003: ClickHouse как аналитическая БД

**Статус:** Принято  
**Решение:** ClickHouse для хранения событий и аналитических агрегатов.  
**Обоснование:** Колоночное хранилище, быстрые GROUP BY, нативная интеграция с Grafana, HTTP API.

---

## ADR-004: Airflow как оркестратор

**Статус:** Принято  
**Решение:** Apache Airflow (LocalExecutor) для ETL и потоковых задач.  
**Обоснование:** Широко используется в production, поддерживает DAG, retry, мониторинг.

---

## ADR-005: Streamlit вместо React для Frontend

**Статус:** Принято  
**Решение:** Streamlit как frontend для аналитики.  
**Обоснование:** Быстрая разработка, нативная интеграция с pandas/plotly, не требует JS-экосистемы.

---

## Data Product: academic_performance

**Владелец домена:** Учебный отдел  
**Интерфейс:** ClickHouse таблицы `grades`, `student_features`; REST API через Cube.js  
**SLA:** Данные обновляются ежедневно к 08:00  
**Метрики качества:**
- Полнота: > 95% студентов имеют оценки за семестр
- Точность: оценки в диапазоне [1.0, 5.0]
- Уникальность: нет дубликатов по (student_id, subject, date)
